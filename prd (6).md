# 产品需求文档（PRD）
## 心理分析辅助评估工具 · 初始版本

---

## 一、项目背景与目标

通过人工智能实现对心理状态的辅助分析，利用自然语言处理与神经网络对文本内容进行分类判断，输出结构化情绪分析结果，作为专业心理咨询师自身判断的对照与补充。

本工具定位为**专业辅助工具**，AI 分析结果仅作参考，不替代专业心理评估。

---

## 二、落地策略（分阶段）

初始版本不等待分类模型训练完成，优先用语言模型跑通全流程，分阶段递进迭代。

### 第一阶段：语言模型全流程（初始版本）

语言模型同时承担分类和解读两个任务，通过 Prompt 设计输出结构化情绪标签加 REBT 风格解读。系统先跑起来，积累真实使用数据和批注反馈。

```
文本/语音输入 → 语言模型（分类 + REBT 解读）→ 结构化标签 + 自然语言解读输出
```

### 第二阶段：引入分类模型

用第一阶段积累的真实标注数据训练专门的分类模型，替换语言模型承担的分类部分。

```
文本/语音输入 → 分类模型 → 结构化情绪标签 → 语言模型（REBT 解读）→ 自然语言解读输出
```

### 第三阶段：稳定运行

分类模型专注结构化判断，语言模型专注解读生成，两者分工明确，持续用批注数据迭代优化。

### 架构预留说明

初始版本系统设计上需预留分类模型接入的接口，确保第二阶段替换时不需要重构整个系统。

---

## 三、技术路线

### 核心架构

```
文本/语音输入 → 分类模型 → 结构化情绪标签 → 语言模型（埃利斯风格 Skills）→ 自然语言解读输出
```

### 模型选型

- **分类模型**：基于 Chinese-BERT-wwm-ext（哈工大版本）进行微调，适用于中文心理文本分类任务
- **语言模型**：通过 API 调用大语言模型，结合现有埃利斯（REBT）风格 Skills 生成自然语言解读
- **两者解耦**，分开开发与优化，最终拼接为完整流程

### 语音输入处理

- 语音经 ASR（自动语音识别）转为文字后进入分析流程
- 后期可考虑声学特征（语调、语速、停顿等）与文本并行分析，提升情绪强度判断精度

---

## 四、分析能力边界

### 当前技术可实现的层次

| 层次 | 内容 | 可行性 |
|------|------|--------|
| 情绪类别与强度识别 | 焦虑、抑郁、愤怒、积极、中性等 | 成熟，准确率 85-90%+ |
| 多情绪复杂性识别 | 主次情绪关系、矛盾情绪并存 | 可行 |
| 认知模式识别 | 灾难化思维、全或无思维、反刍思维、自我归因 vs 外部归因等 | 基本可行，依赖标注质量 |
| 情绪指向识别 | 针对自身、他人或特定情境 | 可行 |
| 情绪强度与持续感区分 | 区分"有点烦"与"完全崩溃" | 可行 |

### 当前不纳入范围

- 跨文本的深层人格特质推断
- 细微防御机制识别（如理智化、否认）
- 成因推断与深度归因分析

---

## 五、输出形式

采用**结构化标签 + 自然语言解读**双轨输出：

- **结构化标签**：由分类模型输出，包含情绪类别、强度、认知模式、情绪指向、风险等级等字段
- **自然语言解读**：将结构化结果传入语言模型，按埃利斯 REBT 框架生成可读性解读文本
- **置信度显示**：附带本次判断的置信度，辅助使用者判断结果可信程度

---

## 六、核心功能清单（初始版本）

### 输入模块
- 文本输入入口
- 语音输入入口（ASR 自动转文字）

### 分析与输出模块
- 结构化情绪分析结果展示（情绪类别、强度、认知模式、情绪指向）
- 埃利斯（REBT）风格自然语言解读输出
- 置信度显示

### 风险预警模块
- 自伤、自杀相关语言信号的独立识别与优先级标记
- 独立通道处理，不混入普通情绪分析流程

### 记录与管理模块
- 来访者基本档案管理（编号/代号制）
- 每次分析结果保存
- 时间轴视图：纵向呈现同一来访者情绪与认知模式的历史变化趋势

### 反馈与批注模块（核心功能）
- 使用者对 AI 输出内容进行文字批注
- 星级评价（量化反馈）
- 结构化分歧标注（如"情绪识别准确但认知模式判断有偏差"）
- 批注数据自动积累，作为后期模型迭代的训练反馈来源

## 七、语料库方案

### 建设思路

分三个阶段递进，每个阶段都有明确的数据来源：

**第一阶段：冷启动**
- CPED（中文心理情感对话数据集）作为底层支撑，给模型建立基础的中文情感理解能力
- 结合现有埃利斯系列书籍和公开案例构建初始领域语料

**第二阶段：领域微调**
- 基于书籍和案例语料对模型进行 REBT 框架下的领域微调
- 系统运行后通过使用者批注反馈持续积累真实场景数据

**第三阶段：自建语料库**
- 积累足够的真实使用数据后，引入自建语料库做深度迭代

### 初始语料资源清单

**书籍资源（埃利斯系列）**
- 《无条件接纳自己》
- 《拆除你的情绪地雷》
- 《理性情绪》
- 《控制焦虑》
- 《我的情绪为何总被他人左右》
- 《控制愤怒》

**案例与公开资源**
- 埃利斯学院公开案例（优先级最高，语言最接近真实咨询场景）
- 互联网公开的埃利斯相关内容（需筛选，质量参差不齐）

### 语料处理方式

- 书籍和案例文本提取后需改写泛化，保留语言模式和情绪特征，不直接使用原文，规避版权风险
- 使用者负责书单和标注维度的提供，以及对生成语料的专业审核
- 互联网内容做最后补充，不作为核心语料来源

### 覆盖范围说明

现有资源对 REBT 框架下的焦虑、愤怒、自我否定、情绪被他人左右等核心情绪维度覆盖充分。后期扩展多咨询师框架时，需补充对应理论框架的专项语料。

### 技术选型说明

- **不采用 RNN/LSTM**：Transformer 架构（BERT）对长距离语义依赖的捕捉能力更强，更适合心理文本中依赖上下文的情绪信号识别
- **微调而非从零训练**：基于 Chinese-BERT-wwm-ext 做领域微调，数据门槛低，迭代灵活
- **注意边缘案例覆盖**：否定句、反问句、轻描淡写的表达容易被误判，标注数据需专门覆盖这类样本

## 九、本地数据存储方案

### 存储格式

采用 **JSON 格式**存储每次分析记录。分析结果本身为结构化数据（情绪标签、置信度、认知模式、解读文本、批注、星级评价等），JSON 格式读写方便，且可直接用于后期模型训练，无需额外转换。

### 目录结构

按来访者编号建立独立文件夹，每个文件夹内按时间顺序存放每次分析记录：

```
/data
  /client_001
    2026-05-04_session1.json
    2026-05-10_session2.json
  /client_002
    ...
```

### 实现方式

网页端连接本地运行的轻量后端服务，数据直接写入本机文件系统，不经过云端。这种方式兼顾了网页端开发的灵活性，同时确保敏感数据不离开本地设备。

## 十、隐私与数据安全

- 来访者数据存储与传输全程加密
- 优先考虑本地存储，避免敏感数据上传云端
- 来访者身份去标识化，使用编号替代真实信息，映射关系单独管理
- 语音原始文件处理完成后不长期保留（声纹属生物特征）
- 使用前需获取来访者知情同意（AI 辅助分析告知）
- 用于模型训练的数据须彻底脱敏，不含任何可识别身份信息
- 初始阶段建议使用模拟数据或匿名样本测试，真实数据在隐私机制完善后接入

## 十一、产品形态

### 初始版本：网页端

- 适合桌面端专业使用场景，信息密度与布局灵活性优于 App
- 开发与迭代成本低，便于早期快速验证核心流程

### 界面风格

- 简洁专业工具风格，参考 Notion / Linear 视觉语言
- 中性色为主，避免高饱和度颜色，保持视觉平静感
- 左右分栏布局：左侧为输入区与来访者信息，右侧为分析结果与解读输出
- 批注功能嵌入结果区域，无需页面跳转
- 时间轴作为独立视图切换
- 字体与间距宽松，降低长时间使用的视觉疲劳

---

## 十二、关键风险与注意事项

- AI 输出仅作对照参考，使用者为主体判断者
- 建议使用者在做出自身判断后再查看 AI 结果，避免锚定效应
- 自然语言解读需通过 Prompt 约束，防止输出类似诊断结论的内容
- 数据标注质量是整个系统准确性的天花板，需专业人员参与
